# 0001. 경량 pnpm 모노레포 채택

- 상태: 채택(Accepted)
- 날짜: 2026-06-24
- 맥락(앱/모듈): shared

## 배경 / 문제
상품 앱과 회원가입 앱은 **별개의 프로젝트**지만 제출은 GitHub 저장소 1개로 한다. 두 앱은 디자인 토큰,
react-query 설정, http 래퍼, cookie 유틸, 로딩/에러 UI 등 적지 않은 공통 관심사를 가진다.

## 결정
pnpm 워크스페이스 기반의 **경량 모노레포**로 구성한다. `apps/{products,signup}` + `packages/{config,ui,lib}`.
무거운 도구(Turborepo/Nx/원격캐시/changesets)는 도입하지 않는다.

## 고려한 대안
- **별개 레포 2개** — 가장 독립적이나 제출 URL 2개, 공통 코드·.claude 설정 중복, 협업 추적 분산.
- **한 레포 + 독립 폴더(워크스페이스 없음)** — 단순하나 config·테마·유틸이 중복되고 재사용을 보여줄 수 없음.
- **무거운 모노레포(Turbo/Nx)** — 앱 2개에는 명백한 오버스펙. 빌드 그래프/캐시의 이득이 비용보다 작다.

## 트레이드오프
- 얻음: 설치 1회·버전 일관성·공통 패키지 재사용·평가 항목(구조/확장성/재사용성)에 직접 부합.
- 포기: 완전한 앱 간 독립성. 단, 두 앱은 서로를 import 하지 않고 `packages/*` 만 공유해 결합도를 낮췄다.

## 영향
- `pnpm-workspace.yaml`, 루트 `package.json`(스크립트), `packages/*`.
- Vercel 은 앱별 프로젝트 2개로 각 `apps/*` 를 root 로 연결한다.
