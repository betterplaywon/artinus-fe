# 0001. 단일 패키지 채택 (모노레포 철회)

- 상태: 채택(Accepted)
- 날짜: 2026-06-24
- 맥락(앱/모듈): shared

## 배경 / 문제
초기엔 상품/회원가입 두 앱을 pnpm 워크스페이스 모노레포(`apps/*` + `packages/*`)로 구성했다.
이후 **범위가 "로그인 페이지 1개"로 축소**되었다.

## 결정
모노레포를 접고 **단일 pnpm 패키지**(워크스페이스 아님)로 재구성한다. 공유 코드(`lib`/`ui`)는
별도 패키지가 아니라 `src/lib`·`src/ui` 폴더로 평탄화한다.

## 고려한 대안
- **모노레포 유지** — 앱이 하나면 공유할 상대가 없어 `packages/*` 간접 계층이 순수 비용. 워크스페이스
  프로토콜·잠금파일 부트스트랩·패키지 간 tsconfig extends 문제(과거 우리를 실제로 물었던 비용)만 남는다.
- **모노레포 유지 + apps/login 만 추가** — 구조는 그대로 둔 채 앱만 단순화. 그래도 워크스페이스 오버헤드 잔존.

## 트레이드오프
- 얻음: 범위에 비례한 단순함. 빌드/설정/멘탈모델 축소. 라우터도 불필요(단일 페이지)해져 의존성 추가 감소.
- 포기: "공유 패키지 설계"라는 어필 포인트. 단, 단일 페이지를 모노레포로 감싸는 것 자체가 오버엔지니어링
  신호이며, 모노레포 작업 이력은 git 히스토리(PR #1~#5)에 보존된다.

## 영향
- `apps/`·`packages/`·`pnpm-workspace.yaml` 제거. 루트 단일 `package.json`/`tsconfig.json`/`vite.config.ts`.
- `@artinus/lib`·`@artinus/ui` import → `@/lib`·`@/ui` 경로 별칭.
